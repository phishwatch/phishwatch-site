PhishWatch

Privacy Policy

Last updated: February 24, 2026

Overview

PhishWatch is a Chrome browser extension that detects browser-native phishing attacks in real time. We are committed to minimal data collection and transparency about what information we handle.

PhishWatch performs local runtime checks in your browser first. Cloud-based risk scoring is only triggered when an outbound click shows risk indicators — such as suspicious URL structure, ClickFix patterns, or credential flow anomalies. Most normal navigation does not trigger cloud checks.

What We Collect

When a cloud check is triggered

When local analysis detects risk indicators on an outbound link click, the following data is sent to our backend for risk assessment:

  • Destination URL — the URL you are navigating to (required for risk assessment)
  • Signal IDs and severity levels — e.g., "instruction_to_execute_recent_copy", "high"
  • Timing metadata — timestamps and time-deltas between events (milliseconds)
  • Boolean flags and field types — e.g., paste_blocked: true, field_type: "password"

What we NEVER collect

  • Page content or DOM structure — some checks inspect page text locally (on-device), but no page content is ever transmitted
  • Form fields, passwords, or credentials — zero form data of any kind
  • Clipboard contents — may be inspected locally for ClickFix/ConsentFix patterns, but never transmitted or stored
  • Cookies or session tokens — no session data, no browsing history
  • User identifiers or tracking data — no account linkage, no persistent user ID, no analytics
Allowlist-based sanitization: Data transmission is controlled by an allowlist function (sanitizeClientSignals()) in the extension's background script. Any field not explicitly on the allowlist is stripped before transmission. Unknown fields fail closed — they are never sent.

Data Processing

Our backend processes scan requests in-memory and returns a risk assessment. PhishWatch does not maintain user accounts, persistent identifiers, or profiles. Standard server logs may record basic request metadata (timestamps and response status). Log retention is controlled by the hosting environment.

Data Storage

PhishWatch stores the following data locally in your browser using chrome.storage.local:

  • User allowlist — domains you have chosen to trust via "Continue anyway"
  • Detection baselines — local novelty detection data for your browsing patterns

This data stays on your device and is never transmitted to our servers.

Third-Party Services

PhishWatch's backend may query domain age information from VirusTotal's public API as part of risk assessment. No user-identifying information is included in these queries — only the destination domain.

Permissions

PhishWatch requests the following Chrome permissions:

  • scripting — required to inject the content script that intercepts outbound link clicks for analysis
  • storage — required to store your local allowlist and detection baselines
  • host_permissions (api.phishwatch.io) — required to send flagged URLs to our backend for server-side risk analysis

Security

PhishWatch is built on Manifest V3 with a strict permissions model. The extension uses no remote code execution (eval()), no dynamic script injection, and is CSP-compatible. All communication with our backend uses HTTPS.

Your Control

PhishWatch uses a fail-open design: when detection is uncertain, browsing continues without interruption. All warnings are overridable — you can always choose "Continue anyway" to proceed. You can uninstall PhishWatch at any time through Chrome's extension manager, which removes all locally stored data.

Children's Privacy

PhishWatch does not knowingly collect personal information from children under 13. The extension does not collect personal information from any user.
