PhishWatch

Privacy Policy

Last updated: March 24, 2026

Short version: PhishWatch performs local analysis in your browser first. When a cloud check is triggered, it sends the destination domain name and sanitised signal metadata — not the full URL, not page content, not credentials, not clipboard contents. Telemetry is disabled by default and requires your explicit opt-in. Data is stored in Frankfurt, Germany (EU) and automatically deleted after 90 days.

1. Who we are

PhishWatch is operated by Deborshi Seal, Berlin, Germany. For privacy questions, contact security@phishwatch.io. PhishWatch is registered as a sole trader operating under German law.

2. What PhishWatch does

PhishWatch is a Chrome browser extension that detects browser-native phishing attacks at the moment of user interaction — intercepting at the click, redirect, and fake login stages, before credentials are exposed. All detection runs locally in your browser first. Cloud-based analysis is only triggered when local signals indicate risk.

3. Data we transmit during cloud analysis

When local analysis detects risk indicators, the following data is sent to our backend (api.phishwatch.io) for risk assessment:

Data elementWhat it is
Domain nameThe registered domain of the URL being analysed — hostname only. No path, query string, or fragment. Example: suspicious-login.com
Signal IDsIdentifiers of phishing signals that fired locally — e.g. cross_origin_credential_post. No signal content, no page text.
Signal metadataBoolean flags and severity levels — e.g. has_password_field: true, risk_band: "high". All fields sanitised via allowlist before transmission.
Extension versionVersion string of your installed PhishWatch extension — e.g. 3.0
Session IDA randomly generated UUID created fresh for each browser tab session, held in memory only. Not persisted to disk. Not linked to your identity.
TimestampWhen the event occurred (ISO 8601 format).
Allowlist-based sanitisation: Transmission is controlled by sanitizeClientSignals() in the extension's background script. Any field not explicitly on the allowlist is stripped before transmission. Unknown fields fail closed — they are never sent.

5. Telemetry — opt-in only

PhishWatch includes an optional telemetry system that records anonymised threat detection events. Telemetry is disabled by default. You must explicitly opt in via the extension popup before any events are stored.

When telemetry opt-in is enabled, the following event types may be recorded:

Event typeWhen it fires
scanEach time a URL is assessed by PhishWatch
warningWhen an overlay warning is shown to you
decisionWhen you act on a warning (proceed, cancel, or report false positive)
session_incidentWhen a redirect chain and a cross-origin credential form are detected in the same browser tab session — a correlated attack sequence
credential_interactionWhen a login form on the page would submit credentials to a different domain than the one shown in your browser

All telemetry events carry only the data fields listed in Section 3. No credentials, full URLs, page content, or user identifiers are ever included in telemetry events.

Telemetry storage and retention

Telemetry events are stored on a Postgres database hosted by Render Services, Inc. in Frankfurt, Germany (EU). Events are automatically deleted after 90 days. No data is transferred outside the European Economic Area for storage purposes.

6. Local storage

PhishWatch stores the following data locally in your browser using chrome.storage.local:

  • User allowlist — domains you have chosen to trust via "Continue anyway"
  • Telemetry opt-in preference — your choice of whether to enable telemetry
  • Session state — in-memory only, cleared when the tab closes

This data stays on your device and is never transmitted to our servers.

7. Third-party services

Domain age checking (RDAP/WHOIS): PhishWatch's backend queries publicly available domain registration data to detect recently registered domains. For most domains, this uses RDAP (Registration Data Access Protocol) — a public ICANN-maintained service requiring no API key. For German (.de), Austrian (.at), and Swiss (.ch) domains, a direct WHOIS query is sent to the relevant national registry (DENIC, nic.at, or SWITCH). Only the domain name is sent — no session IDs, signal data, or user information. These are public infrastructure services, not commercial third parties. If the query fails or returns no data, this check is skipped and navigation proceeds normally.

Render Services, Inc.: Our backend API and database are hosted on Render's managed platform in Frankfurt, Germany (EU). Render processes data on our behalf under a data processing agreement. The database is encrypted at rest (AES-256). For enterprise and compliance details, see our GDPR Article 32 technical measures document.

No other third-party services receive any data from PhishWatch.

8. Chrome permissions

PhishWatch requests the following Chrome permissions:

  • scripting — required to inject the content script that analyses browser mechanics at runtime
  • storage — required to store your local allowlist and preferences
  • host_permissions (api.phishwatch.io) — required to send domain names and signal metadata to our backend for risk analysis

PhishWatch does not request: tabs, history, webNavigation, webRequest, cookies, or any other browsing data permissions.

9. Security

PhishWatch is built on Chrome Manifest V3 with a strict permissions model. The extension uses no remote code execution (eval()), no dynamic script injection, and is CSP-compatible. All communication with our backend uses HTTPS/TLS 1.2+. All secrets are stored as environment variables and never committed to code or logs.

10. Your rights and controls

Opt out: You can disable telemetry at any time via the extension popup. Detection continues normally without telemetry.

Uninstall: Uninstalling PhishWatch via Chrome's extension manager removes all locally stored data immediately.

Data deletion: To request deletion of any stored telemetry events, contact security@phishwatch.io. We will action deletion requests within 30 days.

Data portability: Because session IDs are randomly generated UUIDs with no link to your identity, we cannot extract records attributable to a specific individual. We can delete all records associated with your organisation's domain on request.

Override: PhishWatch uses a fail-open design. All warnings are overridable — you can always choose "Continue anyway."
